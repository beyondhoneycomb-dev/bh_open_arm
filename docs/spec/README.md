# OpenArm v2.0 피지컬 AI 플랫폼 — 기능 명세서

OpenArm v2.0 양팔 로봇으로 **텔레오퍼레이션 → 데이터 수집 → 학습 → 추론 → 평가** 전 과정을 하나의 GUI 플랫폼에서 수행하기 위한 기능 명세다.

**런타임은 LeRobot이다.** OpenArm은 LeRobot의 1급 로봇(`openarm_follower` / `bi_openarm_follower`)이다. 우리가 만드는 것은 **VR/KER 텔레오퍼레이터 + 안전 계층 + 실기 평가 하네스 + 통합 GUI(웹 SPA + 헤드리스 FastAPI 백엔드)**다. 안전 계층·캘리브레이션·이중추론은 사용자의 기존 작업물 `bh_indy7_LeRobot`/`bh_rebot_LeRobot`에서 **백엔드 로직 패턴을 이식**한다(PyQt6 UI는 웹으로 대체, 수치는 v2).

**단, LeRobot의 openarm 통합은 v1 시대(PR #2795, 2026-01-28)에 작성됐다** — 리밋·게인·그리퍼가 v1-era 값이므로 **전부 OpenArm v2.0 정본으로 교체**했다(00 §3.1 판별 매트릭스).

**이 묶음은 기획서가 아니다.** 로드맵·일정·비용·시장은 없다. *"시스템이 무엇을 할 수 있어야 하는가"*와 그 입출력·파라미터·제약·인터페이스 계약만 기술한다.

## 이 명세가 선 사실 기반

- **25개 리서치 에이전트**가 1차 소스(GitHub raw 파일, 공식 문서, 논문)를 직접 읽어 노트를 작성.
- **적대적 완결성 비평** → 갭 식별 → 보완 리서치.
- **반증 검증 2차**: LeRobot **v0.6.0 로컬 스냅샷**(공개 최신; 스냅샷 `pyproject.toml`이 자칭한 `0.6.1`은 PyPI·태그 없는 미출시 유령 버전이라 v0.6.0 기준으로 읽는다)과 `enactic/*` 라이브 소스를 **직접 열어** 8개 핵심 주장을 재판정 → **3건 REFUTED, 2건 조건부**. 이 결과가 **런타임 전제를 dora에서 LeRobot으로 뒤집었다.**
- **반증 검증 3차 (v3)**: LeRobot의 openarm이 **v1 시대(PR #2795, 2026-01-28)에 작성**됐음을 커밋 타임라인으로 확정하고, 사용자의 기존 작업물 두 참조 레포(`bh_indy7_LeRobot`/`bh_rebot_LeRobot`)를 **직접 열어** 안전·캘리브레이션·GUI 패턴을 대조했다. **v1(리밋·게인·그리퍼) / reBot(150°/s·6관절 등 타 로봇 수치) / 오버엔지니어링(GUI 전송 계층) 3축 오염**을 걸러 v2 정본으로 교체.
- **명세 감사**: FR/NFR ID 충돌 0건, 문서 간 모순 식별 → 해소.

### 🔴 반증되어 배제된 것 — 다시 명세에 넣지 마라

근거는 [16-미해결-이슈 §10](16-미해결-이슈.md)에 있다.

**런타임 전제 (v1 → v2 전면 교체)**

| v1에서 믿었던 것 (틀림) | 실제 |
|---|---|
| **dora-rs가 정본 런타임** | **LeRobot 중심.** dora는 안 쓴다. 하위 lib(`openarm_control`, `openarm_driver`, `openarm_ker`)은 쓴다. dora 노드는 `node.send_output` 수준의 **얇은 래퍼**이고 로직은 순수 Python이다 |
| `dataflow-vr.yaml` vs `dataflow_bridge_ros2_vr.yaml` 중 **택일** | **둘 다 안 쓴다** |
| ROS 2 `ros2_control` / `OpenArmHW` / MoveIt이 경로에 있다 | **경로에 없다** |
| **OpenArm Dataset v0.3.0이 정본 포맷**, LeRobot으로 변환 | **LeRobot 포맷이 처음부터 정본.** `openarm_dataset` pip는 **라이터가 없다**(리더/컨버터 전용 — src 트리에 writer 모듈 부재) |
| `qtorque`가 별도 채널 | **`observation.state`로 평탄화** (단완 **24차원**, 양완 **48차원**). `observation.effort`는 LeRobot에 **존재하지 않는다**(grep 0건) |
| MoveIt KDL이 부적합하니 **IK 솔버를 교체**해야 함 | **`openarm_control`(mink + MuJoCo + daqp)을 쓴다.** 실측 양팔 **0.345 ms / p95 0.355 ms** → 60 Hz 예산의 **2%**. **IK는 병목이 아니다** |
| **Isaac Lab은 정체/archived** | **살아있다.** archived된 것은 `openarm_simulation`이고, 이유가 정체가 아니라 **시뮬레이터별 저장소로 분화**(README: *"we've created separated repositories for each simulator"*). `openarm_isaac_lab` = Isaac Sim **5.1.0** / Isaac Lab **2.3.0** |
| 프로세스 간 CAN 소유권 중재가 1급 요구사항 | **축소되지만 사라지지 않는다.** LeRobot Python API를 단일 프로세스에 임베드하면 모드 간 배타성은 내부에서 해결된다. 단 커널은 배타성을 강제하지 않으므로 **`flock`(외부 침입자 방어) + 모드 상태머신은 유지** |
| USB-CAN 어댑터가 **USB Full-Speed 12 Mbps** (`gs_usb` 가정) | **틀렸다.** Pibiger SavvyCAN-FD-X2는 **High-Speed USB 2.0**, in-tree 커널 드라이버. USB 링크 대역폭은 **병목이 아니다**(~2–4%). 이 정정은 지연 예산을 **유리한 방향**으로 바꾼다 |
| 바이래터럴은 `openarm_teleop`(C++) **단독 경로뿐** | **거짓.** 버스 레벨 tau 채널은 이미 뚫려 있다(`damiao.py:492-502`). 막힌 건 `send_action`의 tau=0 하드코딩 **하나뿐**. **진짜 제약은 주파수(500 Hz)이고, LeRobot Python 루프의 실측 사이클 타임은 [미측정]이다** |

**수치·사실 (v1에서 이미 반증, 유지)**

| 흔한 오류 | 실제 |
|---|---|
| 그리퍼 = v1 링키지 병렬(88 mm) / 파지력 상한 1.0 Nm(로드셀 측정) | **v2는 pinch(−45°–0°) + 인핸드캠 + POS_FORCE.** `gripper_posforce_limits`의 2번째 값은 **무차원 torque_pu ∈ [0,1]**(내부 /4.5 스케일), **"1.0 Nm"이 아니다** — 속도는 **DM4310 vMax 30 rad/s 이하로 클램프**(50 rad/s는 초과)하고 Nm/N 환산은 실기 파지력 캘리브 후에만 표기한다 `[개정 2026-07-15]`. 스트로크는 **엔드포인트 rad 캡처**(`capture_gripper_endpoint`)로 영속 |
| 텔레옵 지연 **≤150 ms** 하드 상한 | 근거 논문이 **CARLA 차량 차선유지** 실험이다. 매니퓰레이션이 아니다 |
| `actions_per_chunk = 50` | **기본값 없는 필수 인자.** `chunk_size_threshold`는 0.7이 아니라 **0.5** |
| `friction.yaml`에 마찰 파라미터 | **0바이트 빈 파일** |
| OpenArm에 VR 코드가 없다 | **있다.** `dora-openarm-vr`(UDP:5006 프로토콜 **완전 공개**), `dora-openarm-webxr`(WebXR, HTTPS:8443) 실재. **v2 실기 VR 동작**(Quest 3) |
| 카메라 = 고정 슬롯(RealSense, 또는 Arducam+ZED SBS 뎁스) | **소프트웨어 계약은 빈 dict**(`cameras=field(default_factory=dict)`, 전량 사용자 설정). v2 정본 경로는 **RealSense 뎁스 + OpenCV 웹캠(로지텍) 혼합**(뎁스는 LeRobot 0.6.x 필요). Arducam B0495+ZED mini는 **Cell 참조 하드웨어**지 우리 계약이 아니다 |
| PD 게인 240 계열이 v2.0 기본값 | **v1 유래.** v2.0 자산 등장보다 **10개월 앞선** 값. 정본은 **compliant=70(공통)/stiff=230(v2)**, 240=LeRobot 독자 비정본 |
| KER은 M5Stack | **미검증.** VID `0x303A`는 **Espressif(ESP32-S3)**다. "ESP32-S3 기반 USB 인코더 모듈"이라고 써라 |

## 무엇이 확정됐고, 무엇이 남았는가

전체는 [16-미해결-이슈](16-미해결-이슈.md). **v3에서 결정 등록부는 대부분 닫혔다.**

**✅ 확정 (v2 정본)** — 16 §4~§8·§10:
- **안전 게이트**: `M-1` 디스크 SoT + 관절 zero만 모터 NV / `M-2` E-Stop=전원 차단→CAN 끊김 / `M-3` 낙하 게이트 무시 / `M-4` comm-loss 워치독(하드 상한 안 박음) / `M-20` connect auto set_zero **오버라이드**
- **물리 게이트**: `D-3` v2 URDF rad 리밋 / `D-4` compliant 70·stiff 230 / `D-5` v2 pinch + POS_FORCE + 엔드포인트 캡처
