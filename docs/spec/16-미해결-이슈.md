# 16. 미해결 이슈 — 결정 등록부 (v3: 대부분 닫힘)

## 1. 이 문서의 상태

`01`~`15`의 모든 `[결정필요]` / `[미확인]` 항목을 **중복 제거하여** 한곳에 모은 것이다. 이 문서가 **유일한 결정 등록부**이며, 각 문서의 §5는 여기를 참조한다.

> ⚠️ **v3에서 등록부는 대부분 닫혔다.** LeRobot의 openarm 통합이 **v1 시대(PR #2795, 2026-01-28)에 작성**됐음을 커밋 타임라인으로 확정했고(→ `00` §3.1), 리밋·게인·그리퍼를 **OpenArm v2.0 정본으로 전면 교체**했다. 안전·물리·데이터·아키텍처 게이트가 **전부 확정**됐고, 남은 것은 **소수의 비차단 자체검증**(§3)뿐이다. 옛 등록부를 참조하고 있다면 **이 문서가 이긴다.**

**실기 전원 인가와 데이터 수집을 막는 미결은 없다.** 물리 게이트(리밋·게인·그리퍼)와 안전 게이트가 닫혔으므로, 이 문서의 톤은 "무엇을 결정해야 하나"에서 **"무엇이 v2로 확정됐나(§4~§10) + 남은 자체검증(§3)"**으로 바뀐다.

## 2. 두 종류를 구분한다

| 종류 | 뜻 | 닫는 방법 |
|---|---|---|
| **`D-n` 결정** | **충돌하는 사실이 전부 실재**한다. 답이 여러 개라서 사람이 **골라야** 한다. | 사람이 결정 |
| **`M-n` 측정** | 답을 **아직 모른다**. 1차 소스에 없다. | 실기에서 **측정** |

`M-n`은 대부분 `D-n`의 입력이다. v3에서 두 종류 모두 대부분 닫혔다.

## 3. 남은 것 — 비차단 자체검증

**게이트는 전부 닫혔다.** 아래는 진행을 막지 않는(비차단) 확인 항목이며, 실패해도 대체 경로가 있거나 하드 상한을 박지 않는다.

## 4. 안전 게이트 — 전부 확정 (실기 전원 인가 조건 충족)

OpenArm에는 **홀딩 브레이크가 없다.** 인에이블이 이탈하면 토크가 0이 되고 팔은 **중력으로 떨어진다.** 공식 Safety Guide: *"if power is lost due to an emergency stop, the load being held will fall rapidly."* 아래 게이트가 닫혀 낙하·영점 거동이 예측 가능해졌다.

| ID | 확정 | 근거 |
|---|---|---|
| **M-1** | **디스크 JSON = SoT** (signs · 그리퍼 open/close rad · captured 플래그 · timestamps, 원자적 tmp→fsync→os.replace). **단, 관절 zero만은 모터 NV**(`0xFE` set_zero)에 산다 — 디스크 오프셋으로 표현 불가. `0xAA` flash store는 firmware별 신뢰 불가 → **미의존** | 참조 레포 `save_calibration_atomic`, Damiao registers, `00` §3.2 |
| **M-2** | **E-Stop = 모터 전원 자체 차단 → CAN도 끊긴다.** "정지 후 상태를 읽어 복구"는 불가능하다는 전제로 설계 | 공식 Safety Guide, `12` §E-Stop |
| **M-3** | **낙하 속도 게이트 제거** — 손상될 수준이 아니다. 낙하 속도 측정 FR/게이트를 삭제한다. 단 낙하 자체는 안전 설계에 유지 — **소프트 스톱(토크로 자세 유지) vs 하드 E-Stop(전원 차단→낙하) 분리는 그대로** | 사용자 결정(DECISIONS-v3 M-3) |
| **M-4** | **RID 9 = 1단위 50 µs, uint32, comm-loss 시 fail-safe(인에이블 이탈).** 하드 상한 FR로 **안 박고**, **명령 신선도 워치독**만 개념으로 둔다. 공장 기본값은 무관(→ §3) | damiao-motor.jia-xie.com/concept/registers |
| **M-20** | **connect()의 auto set_zero를 오버라이드한다.** `connect()`는 자동 캘리브를 돌리지 않고(`rebot.py:247-249`), bring-up은 `connect_readonly()`(torque-OFF), 영점은 operator가 rest에서 명시적으로 찍는다. SoT=디스크. (LeRobot이 connect마다 `set_zero`하는 이유 = *rest 자세 가정* 재영점; DM zero는 flash 미영속) | `rebot.py:247-259`, `openarm_follower.py:152-153` |

## 10. 이미 닫힌 것 (v1 → v2 런타임 전제 + 수치·사실)

리서치·검증 과정에서 결정되었거나 반증된 것들. **다시 열지 마라.**

### 10.1 런타임 전제 — v1에서 뒤집힌 것

| 항목 | 결론 | 근거 |
|---|---|---|
| **D-1: dora가 CAN을 소유하는가, ROS 2가 소유하는가** | 🔴 **둘 다 아니다 — LeRobot이다.** `[확정]` | **OpenArm은 LeRobot 1급 로봇**: `make_robot_from_config` 하드코딩 분기(`robots/utils.py:63-68` → `openarm_follower`, `bi_openarm_follower`). `DamiaoMotorsBus`는 `openarm_can`을 링크하지 않는 독립 python-can 구현. dora 노드는 `node.send_output` 수준의 **얇은 래퍼**다 |

| **정본 데이터 포맷** | **LeRobot 포맷이 처음부터 정본** `[확정]` | `openarm_dataset` pip(0.3.0)는 **라이터가 없다**(리더/컨버터 전용). → 변환 단계가 사라졌다 |
| **`qtorque` 기록이 가능한가** | **가능하다. 그리고 공짜다.** `[확정]` | `openarm_follower.py:90-99` + `:239-246`. 주석 *"in one CAN refresh cycle"* → 추가 CAN 비용 없음. ⚠️ `use_velocity_and_torque` 기본 `False`, `observation.effort` 없음 → `observation.state`로 평탄화(24/48) |

| **USB-CAN 어댑터가 USB Full-Speed(12 Mbps)인가** | **아니다** `[확정]` | Pibiger SavvyCAN-FD-X2 = **High-Speed USB 2.0**, in-tree 커널 드라이버. USB 링크 대역폭은 병목이 아니다(~2–4%) |

## 11. 함정 7개 — 명세 전체가 방어해야 할 것

**전부 "조용한 실패"다.** 예외도, 에러 로그도, 리턴 코드도 나오지 않는다. `00` §4와 동일하며, 여기서는 **어느 요구사항이 이를 막는가**를 명시한다.

| # | 함정 | 결과 | 방어하는 요구사항 |
|---|---|---|---|
| 1 | `use_velocity_and_torque`를 **팔로워·리더 동시** True로 켜지 않음 (`config_openarm_follower.py:71` 기본 `False`) | 팔로워만 True → `build_dataset_frame`이 `values["joint_1.torque"]` 인덱싱 → **record 런타임 KeyError**. 둘 다 False → **조용히 위치-only 8차원 데이터셋** | **`FR-OPS-081`** (단일 스위치로 묶어 동시 강제) |
| 2 | **`--robot.side=left\|right` 미지정** | `joint_limits` 기본값이 **±5°**(`config_openarm_follower.py:107-120`) → **팔이 사실상 안 움직인다** | **`FR-OPS-014`** (side 미지정 기동 거부) |
| 3 | **`--dataset.push_to_hub` 기본이 `True`** | 명시적으로 false 안 하면 **사내 데이터가 HF Hub로 나간다** 🔴 | **`FR-OPS-082`** (기본 False 강제 + 명시적 opt-in + 감사 로그) |
| 4 | **모드 전환에 disconnect/reconnect** | `set_zero_position()` 재호출 = **영점 파괴**(`openarm_follower.py:152-153`) | **`FR-OPS-083`** (세션당 connect 1회) + **`FR-OPS-085`** (CLI 스폰 금지) |
| 5 | **`ip link`로 CAN-FD를 켜지 않음** | python-can socketcan이 `bitrate`를 **무시한다**. 링크가 CAN 2.0인데 `fd=True`로 열면 **조용히 성공하고 통신만 깨진다** | **`FR-OPS-084`** (기동 시 `ip -details link show` 검증, 불일치 시 기동 거부) |
| 6 | **rad ↔ deg 변환 누락 / URDF·MJCF 리밋 ≠ LeRobot 리밋** | `openarm_control` IK는 rad, LeRobot은 deg. MJCF `jnt_range`만 보는 IK가 소프트 리밋 밖 해를 낸다. 게다가 `kinematics.py:220-231`이 `NoSolutionFound` 시 `limits=[]`로 재시도해 **리밋을 버린 해**를 낼 수 있다 | **`FR-OPS-043`**(IK 실패 검출, 무제약 폴백 기본 비활성) + `FR-TEL-*`(어댑터 클램프) + §6 D-3 |
| 7 | **LeRobot 안전장치를 신뢰** | `joint_limits` + `max_relative_target`(기본 OFF)이 전부. **속도/가속/저크 제한·소프트 E-Stop 전부 없다** → 참조 레포에서 이식(`00` §3.5) | **`12` 전체** + `FR-OPS-012`(속도 리밋과 점프 가드 분리) + `FR-OPS-039`(명령 안전 필터) |

## 12. 결정 기록 방법

결정이 내려지면:

1. 이 문서의 해당 항목을 **§4~§10 "확정/닫힌 것"으로 이동**하고 결론과 근거를 적는다.

**결정을 문서에 반영하지 않으면 결정하지 않은 것과 같다.**
