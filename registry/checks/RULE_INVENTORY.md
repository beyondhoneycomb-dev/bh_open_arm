# CI 규칙 실물 대조표 — `WP-BOOT-03` 수용 판정 근거

`WP-BOOT-03`의 계약은 *"이 WP는 규칙을 발명하지 않는다: `06` §5에 없는 검사를 추가하거나
**있는 검사를 빼면 위반**"*이다. 그래서 "무엇이 §5에 실재하는가"가 판정의 전부이며,
이 표가 그 실재 목록이다.

목록은 `docs/plan/06-추적성-레지스트리.md`를 **기계적으로 파싱**해 만들었다 — 규칙 칸과
이름 칸이 함께 있는 표 행만 규칙으로 인정하고, 산문 언급은 세지 않는다. 재생성:

```bash
uv run python -c "
from pathlib import Path
from registry.ingest.markdown import all_tables, plain_text
import re
CI = re.compile(r'\bCI-\d{2}[a-z]?(?:-자기적용)?\b')
for t in all_tables(Path('docs/plan/06-추적성-레지스트리.md')):
    i, n = t.column_index('규칙','항목'), t.column_index('이름')
    if i is None or n is None: continue
    for row in t.rows:
        m = CI.search(plain_text(row[i]))
        if m: print(m.group(0), '|', plain_text(row[n])[:40])
"
```

## 🔴 놓치기 쉬운 두 가지

이 둘이 이 문서가 존재하는 이유다. 둘 다 "빼면 위반"에 해당한다.

### 1. `CI-16`은 표에 없다

§5 표만 훑고 구현하면 **통째로 빠진다.** `CI-16`(그래프 대조)은 §5.6에서 별도 절로
선언되며, `01`에는 *"§1의 '두 번째 진실' 대가를 상쇄하는 유일한 검사"*라고 적혀 있다.
표에 행이 없는 것은 편집상의 사정이지 규칙이 없다는 뜻이 아니다.

### 2. `CI-11b-자기적용`은 하위 조항이 아니라 독립 규칙이다

이름 칸이 따로 있고("CI-11b 자기 코퍼스 green") 자기 수용 기준 ①②③을 갖는다.
요구하는 것은 **검사기를 실제 코퍼스에 돌리는 것**이다:

- ① 위반 픽스처(매니페스트 `gates:` 값과 레지스트리 `gate[]` 값에 맨 `PG-RT-001`)가 **거부**됨
- ② **예외 픽스처 3종**(산문 계열 지칭 · 축약어 표 행 · 이 규칙을 서술하는 칸)이 **통과**함
- ③ 현행 `00`·`04`·`06` 원문이 **green**

`06` §5가 그 이유를 적어 뒀다 — *"돌지 않은 검사기는 문법적으로 존재할 뿐 작동이
증명되지 않았다."*

## 짓는 범위 ≠ 판정 범위 (정정 금지)

| | 범위 | 근거 |
|---|---|---|
| **짓는다** | `CI-01`~`CI-18` 전량 (위 35개) | `WP-BOOT-03` 계약 — 있는 검사를 빼면 위반 |
| **판정한다** | `CI-01`~`CI-17` | `02a` §−2.2 대역 수용 조건 ② |

**두 수를 같게 "정정"하는 것이 결함이다.** `CI-18`의 술어 자신이 대역 수용 게이트 2조건을
참조하므로, 판정에 넣으면 **게이트가 자기를 참조한다.** 게다가 `CI-18`은 `BOOT` 대역 **밖**
WP만 보므로 착지 시점엔 공허참이다 — 판정에 넣어도 아무것도 보증하지 않는다.
